~Variables de bloque y constantes~

var Vs let

En ES se agrega una nueva forma de definir variables usando la parala "let", se diferencia de "var" en que el scope de una variable definida como "let" es, el bloque en el cual se encuentra la variable y no la función.

La variable "var" es considerada como mala practica bedido a que genera variables globlales y let genera variles de bloque


~Constantes~

(const)

Una constante es un tipo "INMUTABLE", NO puede cambiar una vez definida, se usa la paralabra "const" en lugar de "var", al igual que "let" se scope es de bloque, son tipos de sólo lectura y se le debe asignar un valor en el momento de su declaración. Son referencias inmutables, pero sus valores no necesariamente.

¿Cuando se utiliza (conts)?
R= Cuando necesitemos una variable que no cambie durante el flujo del programa.

NOTA: Js permite cambiar propiedades de "objetos" y "arreglos" con "conts".

~Booleans~

En ciencias de informatica, un "boolean es un dato lógico que solo puede tener los valores "true" o "false".

Truthy

En JS, un valor verdadero es un valor que se considera "true/verdadero" cuando es evaluado en un contexto "Booleano". Todos los valores son verdaderos a menos que se definan como falso (es decir, excepto false, 0, "", null, undefine, y NaN).

Ejemplos de valores verdaeros (Truthy):

  • if (true)
  • if ({}) //Objto vacio
  • if ([]) //Un array
  • if (42)
  • if ("foo")
  • if (new Date()) //Fecha
  • if (-42)
  • if (3.14)
  • if (-3.14)
  • if (Infinity)
  • if (-Infinity)

Falsy

Es un valor falso que se considera "false/falso" cuando es evaludo en un contexto "Boleano".

Ejamples de valores falsos (Falsy/Falsey)

  • if (false) //Falso
  • if (null)
  • if (undefined)
  • if (0)
  • if (-0)
  • if (0n)
  • if (NaN)
  • if (""/'') //Cadena de txt vasia
Los operadores lógicos AND, y &&.
  • false && "dog"
  • 0 && "dog"
